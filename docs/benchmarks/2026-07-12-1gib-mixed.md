파일 끝의 exact-size padding은 ASCII다. 생성 corpus는 보고서에 포함하지 않는다.

## 명령

```console
$ target/release/kfind --literal --quiet --no-ignore 'kfind-희귀-검색-표식-7d4b' '<corpus>'
$ rg -F --quiet --no-ignore 'kfind-희귀-검색-표식-7d4b' '<corpus>'
```

두 명령의 종료 코드 1은 고정 literal이 없는 no-match 결과이며 전체 corpus를 scan한다. 각 run은 10회 scan의 합산 시간을 1회당 평균으로 환산했고, 실행 순서는 run마다 교대했다.
