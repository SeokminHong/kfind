## 판정

후보는 canonical full-POS의 raw FP를 2건에서 1건으로 줄이고 raw FN 2건을 유지한다.
Query matrix full-POS는 raw `TP 1,292 / FP 4 / TN 1,292 / FN 4`, contract-adjusted
`TPᶜ 1,296 / FPᶜ 0 / TNᶜ 1,293 / FNᶜ 0`이다. Contract-adjusted precision·recall·F1은
모두 100%다.

Development full-POS raw FN은 기준 15건에서 후보 17건으로 늘었다. 추가된 2건은
`안 나오면`과 `착하다 보니`를 붙여 쓰고 source 표제어도 잘못 정렬한 입력이다. 두 case를
실행 전에 고정한 registry로 계약 분모에서 제외하면 기준과 후보가 모두 `FNᶜ 15`다. 후보는
계약 안의 `빼놓을`과 `물줄기는`를 회수하므로 contract-adjusted 품질은 기준과 같다.

성능 경고선 10%를 넘는 회귀는 없다. Full-POS component 포함 초기화는 5.33%, full-POS
상주 메모리는 2.34% 늘었다. 이 불리한 결과를 채택 결과에 포함한다. Canonical 처리량은
embedded 0.87%, full-POS 1.02% 늘었다.

## Raw와 contract-adjusted

Raw는 corpus gold를 그대로 적용한 관측값이다. Contract-adjusted는 같은 예측에 고정 review
registry만 적용한 제품 계약 값이다. Review가 없는 fixture도 contract-adjusted 값을 생략하지
않고 raw와 같은 값과 review 0건을 기록한다.

Query matrix full-POS의 raw FN 4건은 실행에서 실제로 누락된 결과다. 이 가운데 1건은 gold
span 정렬 오류이고 3건은 비표준 붙여쓰기다. 따라서 네 누락을 숨기지 않고 raw `FN 4`로
보존하되, 제품이 지원하는 표준문 계약 안의 false negative는 `FNᶜ 0`으로 판정한다. Raw FP
4건은 문법 구조로 구분할 수 없는 동형이의 3건과 source에 정렬된 내부 성분 1건이므로
contract positive로 재분류한다.

Test query matrix review 22건은 구현 목표 확인 14건, 기대값 재분류 5건, 비표준 입력 제외
3건이다. Development review 2건은 모두 비표준 입력 제외다.

## 재현
